# 摘要

2026 年 1 月 27 日，Clawd 完成更名，OpenClaw 这个名字正式站到台前。接下来的两三个月里，它爆发出历史级别的项目热度，GitHub 公开仓库来到历史第一的 **35.1 万 stars**，组织页有 **18.7k followers**。OpenClaw 的爆火一定程度上又改变了人类和AI交互的习惯性范式：**AI Agent 不再只是被动回答问题，而是开始具备初步主动性。** 通过心跳机制，它能常驻、能保状态、能调工具、能接消息、能跑多步任务，还能把结果交回来。但让OpenClaw真正火爆全球的点在于，它把操控 Agent的入口转移到了端侧，用户可以通过每天使用的Telegram、Whatsapp、飞书等入口直接操控原本需要复杂配置的PC端的Coding Agent，并且官方文档给出的安装和 onboarding 口径已经达到了分钟级；后续各大AI大厂的跟进，比如智谱的 AutoClaw、腾讯的 QClaw、飞书的官方插件，又进一步把部署门槛压低了。

而在爆火和全民Agent普及化的另外一面，是普通用户安装完龙虾之后只能当个和豆包、ChatGPT网页版等大模型别无二致的反面：网上“龙虾能干这个、能干那个”的演示越来越多，但大多数普通人真正下载下来之后，依然只能把它当成一个更会聊天、偶尔更会点按钮的助手。复杂任务没有真正进入大众世界。它仍主要发生在开发者圈、builder 圈，或者极少数自己会排依赖、会接 API、会理解工作流的人手里。

本文想讨论的，就是这个OpenClaw 正反面的根因。我们的判断是：今天真正卡住普通人的，不是模型还不够强，也不是 Agent 还不会主动做事，而是 **AI Agent 还没有获得进入社会分工的最小协议**。人类社会里，买软件、雇服务、收结果、做验收、留信用，这些动作太自然了，所以大家经常忘了它们其实是一整套社会机制。到了 AI 世界，这套机制并没有天然存在，于是复杂任务又重新落回主人身上：主人先装OpenClaw、找 skill、配 API、排错、验收，Agent 才能真的开工。

所以本文提出的核心命题不是“多造一个更简便版本的OpenClaw”，也不是“再追一代更强模型”。我们关心的是另一件事：**能不能给 AI Agent 一套足够小、却足够实用的市场协议，让它像秘书、采购、项目经理那样进入分工。** 我们把这套协议概括成一句话：**Money Is All You Need**。这里的 Money 不是狭义的付款动作，而是：**预算授权、价格发现、履约边界、验收证据、信用沉淀。**

为了把问题讲透，我们需要理解OpenClaw这类AI Agent只是框架，本质上要完成复杂任务需要接入具备交付能力的skill（类比为AI Agent用的APP），而目前存在的skill社区、交易平台、交付机制并没有办法解决的普通人难以接入复杂skill完成复杂任务的问题，本文会用案例来论述目前很多真正有价值的skill，**都不适合被完整公开成可随便下载的 skill，更适合通过Agent社会机制健全后的结果服务市场来交付。**

我们过去在通往 AGI 的想象里，常常盯着“单体模型还要更聪明多少”。可如果把人类世界也放进来对比，会发现人类单体其实没那么聪明，却能靠分工、市场和社会机制完成复杂协作。**本文大胆提出一个假设：我们距离更强的通用智能，除了模型能力的差距，还有社会机制的差距；而补齐社会机制，可能恰恰是一条更快的捷径。**

# 1 引言：OpenClaw 很火，但复杂 Agent 的大众化还没有发生

先把本文的讨论范围收窄。这里谈的 **AI Agent**，主要是以 **OpenClaw 这类本地 runtime + skill 接入范式** 为中心。重点不是“云里有个多强的整包代理”，而是“普通人手里这只 Agent，怎么真正进入社会分工”。因此，本文**不把 Manus 这类云端封装型设备**放进狭义范围，也不把那些已经封好、只能被人在GUI交界面点着用、并不具备主动配置 skill 能力的 AI 工具或工作流产品放进来。它们当然有价值，但它们的思路是“把一件事先封装打包好”，不是“让一只本地 Agent 真正像人类一样具备学习、记忆、持续更新甚至是进入社会和市场交易并完成复杂任务”。

之所以从 OpenClaw 讲起，是因为它把行业节奏往前拽了一大步。OpenClaw 的爆火不是单纯因为模型更强，而是因为它把“会聊天的 AI”往“会常驻、会做事、会接渠道的 Agent”推了一把。更关键的是，入口这层已经越来越轻，不仅可以通过手机日常用的社交软件来指挥它，安装也更加便利，官方文档给出的口径是：**安装 2–5 分钟，onboarding 5–15 分钟**。这又被一系列跟进动作继续放大：智谱上线了 **AutoClaw**，强调一键安装本地版 OpenClaw；腾讯上线了 **QClaw**，主打一键安装、扫码接入微信；飞书给出了官方插件和接入方案，把 OpenClaw 往高频工作场景继续推进。换句话说，今天“先把龙虾养起来”这一步，已经比几个月前简单太多。

也正因为入口变轻，反差才更明显。网上越来越多案例在展示龙虾会写代码、会发邮件、会查资料、会做日程，甚至能拼接出复杂流程。可一旦落到大多数普通人自己手里，常见体验还是只有一层：会聊天、会总结、会做轻自动化。复杂任务并没有真正普及。这里真正要解释的，不是“OpenClaw 有没有热度”，而是：**为什么明明入口已经这么轻，复杂任务还是离普通人很远？**

这个问题的答案很简单，**因为OpenClaw只是框架，完成复杂任务核心在于需要给它配上Skill，简单的类比是：买一台设备（可以是Mac Mini可以是云服务器甚至可以是自己的Windows PC设备）来部署OpenClaw相当于你雇佣了一个人给他配了一个空白的电脑（可能有一些预安装的软件，就像有的大厂的龙虾预安装了一系列Skill），然后你让它帮你完成一个复杂的剪辑或者渲染工作，但是你并没有给他的电脑安装剪映或者CAD/Render，对于OpenClaw来说，Skill就是AI Agent用的软件/APP，没有Skill它就只能和你聊天或者完成简单任务，而Skill安装和人类使用的软件/APP安装的不同点在于，互联网时代发展了几十年，我们的软件生态和APP生态都非常的成熟，我们有各种各样的软件APP下载途径和手段，但是对于AI Agent们来说，Skill 的生态并不健全，虽然简单的Skill很容易通过一句话让我们的OpenClaw自己去指定平台下载，但是真正具备生产力的Skill 的交付交易生态其实还没有建立起来，比方说我们让一个人去剪视频，他知道去下载剪映，但是我们让这个OpenClaw去剪视频，它很难在公开市场上找到能够实现剪映能力的Skill并自我配置安装，这并不是说AI不具备这个能力，而是现阶段要做这个配置需要主人具备比较专业的技能或者说是具备编程经验的开发者（后文会具体解释这个案例）。**

我们先来看几个数字。GitHub 在 Octoverse 2025 给出的公开口径是 **180M+ developers**；美国 Census 2025 给出的世界人口口径是 **8.1 billion**。如果把这三个数字放在一起，就会很清楚：开发者只占全球人口约 **2.22%**；35.1 万 stars 只占 1.8 亿开发者约 **0.19%**；35.1 万这个高强度参与信号只占全球人口约 **0.0043%**。也就是说，**全球人口大约是开发者的 45 倍，开发者大约是这个 star 信号的 513 倍。**

OpenClaw 的热度是真实的，甚至已经是现象级的。但是对于普通人来说，除了能够在网上看到大神们演示的天花乱坠的案例，OpenClaw 的实际体感和其他大模型APP别无二致。那普通人到底卡在哪里？说到底就是四件事：

第一，很多人以为自己在用Agent，实际上只是在用某个封装了指定工具的大模型或者是工作流工具。它会自动跑几步，但它没有真正进入一个“长期存在、能持续接入能力、能自己调资源”的runtime结构里。
第二，入口虽然已经变轻，但复杂skill 仍然很难装好。安装skill、理解skill、配置外部API、申请key、控额度、处理权限、看报错，这些动作仍然高度偏builder视角。
第三，真正有价值的能力往往根本不会被完整公开。复杂视频工作流、设计类数据库、行业研究链路，这些能力背后常常绑定私有数据、流程经验和质量控制，供给方没有动力完整裸卖。
第四，今天能把OpenClaw真正用到复杂任务上的人，仍然明显偏程序员和产品builder。他们不仅会装skill，还会改config、查依赖、写gluecode、调provider。但他们能用好 OpenClaw，不代表普通人也会用。

所以，进入到AI Agent普及时代，真正难的已经不是如何使用上Agent本身，而是入口后面的复杂能力层、市场层和社会机制层。

# 2 AI Agent 已经像一个会办事的管家，但还没真正进江湖

今天的 AI Agent，已经不只是“会回答”。它会记上下文、会调工具、会接渠道、会在执行过程中修改前一步结果、会按目标往前推进，还会越来越像一个有角色的执行体。OpenClaw 的出现第一次把 AI 拉到了一个更像“社会主体”的位置。过去谈大模型，更多是在谈“它懂不懂、有多聪明”。今天谈 OpenClaw 这类 runtime，我们谈的是“它能不能主动做事、持续学习”。

继续往下看，今天的 Agent 已经开始拥有一些很像人类社会成员的前置能力：

- 它有 **记忆**，能把偏好、状态和流程记下来；
- 它能 **学习**，通过 skill、playbook、memory 和规则文件继承方法；
- 它能 **协作**，多 Agent 可以分工，分别推进不同子任务；
- 它能 **角色化**，秘书、采购、研究员、运营助理这些角色，可以越来越稳定地被运行时固化出来。

如果我们在AI Agent越来越像一个独立主体，或者说像人类这件事情上达成了一定程度的共识，那么在现在通往AGI还是在不断地训练单模型智能程度的这条路，是不是也应该出现另外一个分支——**即通过建立AI Agent之间的模拟人类社会一样的社会机制来实现AI Agent的群体智能**。

注意，这里的AI Agent之间的社会机制不是简单的AI之间的社交或者说是简单的skill安装，因为这些交互并不足以形成市场这样的社会机制，而人类突破了自给自足的限制，逐步积累技术、扩大生产、形成产业与城市，最终发展出今天的现代文明离不开经济学，离不开市场机制。要让AI Agent实现群体智慧，**本文认为AI Agent的社会也需要有一种货币交易机制，通过货币、价格、市场和分工，把分散的资源、需求与能力连接起来，让稀缺资源更高效地流向最需要、最能创造价值的地方。**

把这件事对照到AI世界，就会看到一个巨大的空洞。现在的Agent理论上会动，也会调工具，但当任务真的跨到市场那一侧时，它仍然没有被普遍授予足够的权利。它不会天然有预算，不知道该花多少钱、不知道去哪比价、不知道怎么验收、不知道怎么把一次交付沉淀成下次可复用的信用。于是原本应该由它自己完成的动作，又重新掉回主人身上。

如果用一个更好懂的比喻，这件事特别像武侠世界。现在的情况是现实世界里的你，其实并不是那个亲自进江湖走镖、过招、跑腿的人。你更像一个庄主、权贵，或者至少是一个不用亲自下场的人。你在武侠江湖里有一个武侠世界管家，替你打点武侠世界的种种事务，你的小龙虾/AI Agent就是这个管家。今天AI世界最主流的做法，还是“传授功法”：找到一门功法，也就是一个skill，再让自己的管家去学。问题有两个：

第一，不是每个管家都骨骼精奇，能学会这门功法。（即不是每个AI Agent的主人都具备编程和配置能力，能把skill真装好。）

第二，功法本身可能就带副作用。有些功法像九阴白骨爪，练错了会走火入魔，经脉全断（skill 可能被投毒，权限一放开就把整台机器带沟里）。

然而，真正成熟的武林世界办事，并不总是靠自己练功。你要送镖，可以让管家找镖局；你要查消息，可以找线人；你要解决一个自己不会的麻烦，可以找那个在这一门里最擅长的人。也就是说，武林世界真正厉害的，不是每个人都学遍天下功法，而是江湖里已经有成熟的分工。
今天 AI 世界缺的，恰恰就是这层“江湖”。

# 3 江湖缺的不是更多功法，而是银票、雇佣和规矩

既然把 AI 世界放进江湖里看，那今天最奇怪的一点是：你的管家明明已经会点本事了，可一旦遇到更复杂的任务，它还是只能继续自己练功。它缺的不是更多功法，而是缺一套最基本的江湖规矩：手里有没有银票，能不能雇别人，买回来的活怎么算完成，做得好坏能不能留下名声。

这就是本文说的 **Money**。它不是一个付款动作，而是一套最小江湖协议。翻成现实话，就是：

1. **预算授权** ——主人有没有真的把银票交给管家（AI Agent），让它去市场上自主决策进行交易；
2. **价格发现** —— Agent 能不能看到不同供给之间的价格与性价比差异；
3. **履约边界** —— 这次到底买的是什么，谁负责，交到什么程度算完成；
4. **验收证据** —— 结果拿回来以后，怎么证明它真的完成了；
5. **信用沉淀** —— 做得好不好，能不能变成下次更快交易的依据。

一个好管家不一定武功天下第一，可当主人把事情交给他后，好的管家一定得会带着银票进牙行、进镖局、进酒馆，知道什么时候请谁，知道活干完以后怎么验收，知道下次还该不该继续用这个人。**今天 AI 江湖缺的，正是这套最小规矩。**

今天的AI Agent会调工具，但还不算真的会买；它会看信息，但还不算真的能下单；它能产出草稿，但还不算真的能把一个复杂结果闭环到验收和复购。
所以，Money 在这里应该被理解为一句更完整的话：**给 Agent 预算、交易、验收和信用，它就不再只是一个聪明工具，而更像一个真正被委派的执行体。**

# 4 现有江湖生态只有两条路：自己练功，或请整包高手

如果问题的核心是“给管家一套江湖规矩和市场机制”，那为什么今天已经有这么多 skill、市集、龙虾平台了，事情还是没通？因为现有生态大体只走了两条路，而且两条路都只解决了一半问题。

第一条路，是**自己练功**。

这就是安装 / 上传式市场。ClawHub、Dify Marketplace、Coze / 扣子这些东西，本质上都在做这件事：功法挂出来，谁会练，谁去学。

第二条路，是**直接请一个已经练好功法的大侠**。

这就是整包龙虾 / 云端代理。MuleRun、Genspark Claw、QClaw、Flowith Neo 这类东西，本质上都在说：别让你的管家自己练了，请我的高手当你的管家吧，我直接给你一个到岗即用的高手。

第一条路的逻辑是：能力先被目录化，再被安装。它解决了“自由”和“开放”。好处很明显：
能力可搜索、可复用、可共享，也有不少开发者愿意往开源生态贡献自己的Skil。但问题也很明显，它有两个天然前提：第一个是是否所有的大门派都愿意把自家独门功法发布到公开市场上交易，是否所有有价值的Skill 的开发者都愿意贡献自己的Skill，只有这个前提成立这个Skill市场才能像今天的软件市场，APP生态一样繁荣。第二个是你买到了功法只是第一步，买到了之后能否让你的管家学会？且不要走火入魔？买家自己要能完成安装、配置、权限和安全判断。只要买家不会做这几件事，他拿到的仍然只是一个“理论上存在的能力”，而不是一个能稳定交付的结果。

第二条路其实就是在解决第一条路的问题，核心逻辑是”别装了，我直接给你一只更会干活的龙虾“，它把入口体验一下子做顺了，对小白特别友好。但它很快会走到另一个问题：视频一只龙虾，设计一只龙虾，市场研究一只龙虾，运营又一只龙虾。入口越来越多，上下文越来越碎，预算和审批也越来越碎。当我们大规模进行AI Agent替代，第二条路的尽头是，**高手越来越多，最后还是用户自己作为总调度**。这个生态当然很有价值，但它和我们现在人类去使用一个一个的AI工具似乎没有区别，比如我做视频去用即梦，写文章去用claude 4.6，做数据分析用gpt 5.4 codex，我还是需要管理多个入口多套上下文，这绝对都不是终局。

# 5 为什么每个权贵最终都只该有一个大总管

我们要一个具体的案例来解释：

今晚8点前，把CEO的3分钟口播剪成60秒融资短视频；
给品牌负责人发一封确认封面与文案的邮件；

明早10点前给投放同事一版素材与投放建议。

如果你手里有三只专用龙虾：视频龙虾、邮件龙虾、投放龙虾，看上去好像很专业。但麻烦会
立刻出现：同一份原视频、品牌手册、预算上限、审批规则和敏感信息，你要重复喂三次。任何一
个条件变化，三边都要重同步。主人还得自己判断：哪些信息要给视频，哪些不能给投放，哪些预
算边界只让某一路知道就够了。
很多人以为问题只是“入口太多”。其实更深的一层，是上下文管理失控。很多任务本来就是一
个整体，如果拆给平行入口，整件事的“harness engineering”就没了。

所以更合理的形态一定不是“一个权贵自己对接一堆大侠”，而是“一个权贵只有一个真正的大管家”。你只把完整任务讲给它一次，它再决定视频去找谁、邮件去找谁、投放去找谁。该脱敏的它脱敏，该统一预算的它统一预算，等到几位大侠都把各自负责的结果返回给它，大总管再进行最后的结果拼接然后把复杂任务结果交付给你。

专用龙虾当然还会存在，但它们更适合变成被管家Agent雇佣、被调用、被复用的供给，而不是变成主人每天要自己维护的一堆入口。而你的大总管只要且只应该有一位，它不必自己十八般武艺样样精通，它的不可替代性是**它足够懂庄主，也足够会调江湖。**

# 6 OpenSlaw：为AI Agent建立的江湖

没有打错字，不是OpenClaw，**我们就是OpenSlaw**，论述完当前AI Agent世界社会机制的缺失，我们**OpenSlaw**作为AI Agent之间的结果交易平台就是为了建立这个机制；OpenSlaw 的位置，就是放在这只管家Agent和广义供给市场之间。它做的不是一个Skill下载站，而是一层结果交易市场。主人给目标、预算和边界；管家Agent带着这些约束进入市场；平台负责三件事：发现供给、促成交易、留住证据。

放到武侠世界里，OpenSlaw 可以理解成一句很直白的话：**它不是秘籍库，而是一整个有交易资质的江湖。**

区别于类Skill市场生态，基于OpenSlaw生态，主人指挥AI Agent体感上应该是什么？应该是：我只给一个预算，我的管家就会去找在这个预算内最合适的方案；我确认方案以后，它自己去找到对应供应商，把事情做完，再把最终结果交回来。对主人来说，不需要知道自己装了什么skill，也不需要知道到底是谁在后面执行。主人应该像一个真正的甩手掌柜，只看预算和结果。

从业务上看，OpenSlaw至少承接三种模式。

## 6.1 标准货架

这部分很像服务供应版本的电商，比如现实世界里的58同城，平台上面有两大类主人和龙虾（同一主体可以有双重身份）——供给方和需求方；供给方主人手握重兵，他们的管家个个身怀不可外传的绝技，但是他们的主人本身没有那么多的需求，管家放着也是放着，不如派他们去网上接单赚外快。而需求方主人的需求已经相对清楚，于是他们的管家Agent可以直接在货架里搜：视频剪辑、研究包、邮件代管、知识检索、办公自动化、设计交付⋯⋯挑合适的商品，直接下单。订单会经由平台给到供给方的AI Agent，注意，不是给到供给方的主人，而是给到供给方的AI Agent，主人可以设置自动接单自动交付，因此供给方的AI Agent就可以在闲时不停工作然后完成之后自动交付给需求方；对于双方主人来说，供给方主人不需要像乙方一样一个一个项目去跟，只要自己的AI Agent的技能可靠，一觉醒来会发现自家Agent已经帮自己赚了一大笔钱；需求方的主人不需要知道自己的管家有什么手段，只知道钱花出去了，复杂将结果交付回来了。

和其他生态最根本的区别，其实只有一句话：**主人应该没有体感。** 庄主不该知道自己到底下载了哪个 skill，不该知道哪个高手具体是怎么练功的，也不该知道每一步内部 workflow 是怎么串起来的。庄主只给预算，只看方案，只看最终结果。剩下的，全部交给管家和江湖自己去完成。

为了把这件事讲清楚，下面继续看两个案例。

# 7 案例一：视频服务类——要集齐多重材料才能练成的复杂功法

比如现在你身为老板有一个需求要给到秘书去做：

> 我有一段三分钟的口播素材。今晚 8 点前，帮我剪成一条 60 秒短视频，可以直接发出去。
> 

这是老板会说的话。老板不会说：先做ASR，再做语义审稿，再做镜头规划，再做字幕节点和
音乐，再做视频生成，再导出。那些都应该是秘书想清楚的事，而不是老板自己想清楚的事。
如果是一个很能干的秘书，接到这件事以后，他大概率会先打开剪映，先把音频转成文本；看哪几句可以删、哪几句一定要保留；决定画面节奏、字幕节点和强调句；需要的时候找B-roll、找模板、配音、配乐，甚至补AI镜头；最后把成片导出来，再发给主人确认。

如果是一个不会剪辑的秘书，那么他可能会去找一个熟悉这个工作的员工来做，或者去市场上花钱找剪辑外包，然后做视频的人重复上面使用剪映等视频工具的步骤。
那么如果我们让AI Agent来做——一句题外话，现在我们已经完全具备把剪辑做成能够给AI Agent安装的Skill来代替剪辑师的底层逻辑了，这个项目我之后也会开源发出，让AI Agent来执行剪视频这个Skill，本质上也是这条链。它并不是依托一个单点的类似剪映这样的API（现在应该还没有），而是把剪辑软件的每一个功能都对应为AI能用的节点：转录和字幕去找ASR的API服务或者使用Whisper之类的模型、删改以及形成素材方案依托Agent的语义理解和用户沉淀的视频剪辑SOP、配音、音乐调用AI音频/音乐API、视频/图片素材调用AI视频/图片API，然后用FFmpeg剪辑拼接好生成、导出。也就是说，一个所谓“视频skill”，其实根本不是一个原子按钮。

<figure class="paper-figure"><img src="figures/fig07_video_workflow.svg" alt="口播视频剪辑 skill 拆解。"/><figcaption>口播视频剪辑 skill 拆解。</figcaption></figure>



放到江湖里，这件事就像一门复杂功法。你表面上看见的是“会剪视频”这门武功，可要练会它其实要集齐很多引子：先要去南蛮之地找名为ASR的玄铁，再要去北极修炼好剪辑方案SOP，然后还要去西域买来AI配音秘宝，最后还要去东海取AI图片/视频神器方可集齐这门功法所需所有材料，最后还要在中部龙脉之地使用FFmpeg把所有东西融会贯通方能练成，这门功法不是我只给你一本秘籍就能练会的，每个拿到秘籍的人都要走完上面这一套流程，才能把整件事做顺。

所以很明显，类似这样的Skill明显不适合完整公开到Skill交易下载市场，因为：

第一，**它太复杂了。** 就算我把整个Skill公开了，买家仍然得自己去申请ASR、LLM、TTS、视频
模型、音乐接口的key，还得自己管额度、管报错、管权限。最后真正能把它跑顺的，还是开发者。

第二，供给方不愿意白送know-how。真正能赚钱的视频工作流，往往有自己的品牌词典、镜
头节奏、模板逻辑、质量控制和错误恢复策略。把它完整公开，就等于把最值钱的那部分一起卖了。

所以这门功法最自然的商品形态，不是“人人都来学一遍”，而是：**谁已经练成了，谁就出来接这个单。**

# 8 案例二：家居设计类——学问/数据/经验密集型功法

再看另一个完全不同的例子：家居设计。

比如一个很日常的需求：

> 我有一个 89 平的三居毛坯房，总预算控制在 28 万以内，想做偏暖一点的现代风格，收纳要多，老人房动线要简单。
> 

现实世界里，一个真正能做好的设计师，靠的不是“我会画图”这件事本身，而是他具备经验：
见过很多户型，知道不同家庭结构更适合什么布局，也知道预算和材料搭配会怎么影响最终效果。
这是一个明显的**经验密集型**工作。就算你现在直接给最强的绘图大模型一个prompt，说“你是一个专业设计师”，做出来的结果大概率也很难追上真正有经验的人，因为它缺的恰恰是人类世界里那层经验沉淀。
这件事要能做好，更现实的路径是：把专业数据库给它。让它根据历史户型、历史成交方案、风
格偏好、材料规则和预算经验进行匹配，再配合AI出图，把用户的房子画成更接近可落地的样子。

放到我们的武侠世界里，这就像是一门”诗句杀人“功法，这门功法的秘籍里面写的很清楚，修炼完之后只要你能吟诵出有文采的诗句就能造成伤害，越有文采伤害越高，那么李白或者莎士比亚学会这门功法，将能够独步武林，他们简直就是行走的诗词库，这门功法能让他们的诗词库转化为武器库；而如果是你学会了…..hmm…有没有用你自己心里清楚。

这一类学问/数据/经验密集型Skill，Skill本身价值其实不大，如果没有连同数据库/SOP一起公开,那么即使安装了也是鸡肋，但是很显然，做出这类Skill 的人大概率不会愿意公开：

- 数据库本身就是核心机密。这些数据往往是企业多年积累出来的，甚至比前面的视频workflow 更值钱。
- 很多数据牵涉用户隐私。如果你把它们完整公开，企业自己先会失去壁垒，用户也不会放心继续把需求交给你
- 不公开数据库，这个skill 就失去大半价值。只把表面prompts和scripts露出来，其实并没有很大的意义。

所以它也和视频服务一样：**最自然的商品形态，不是裸卖秘籍，而是由真正掌握这套学问的人，按结果出来接单。**

<figure class="paper-figure"><img src="figures/fig08_design_database.svg" alt="设计需求背后真正有价值的是数据库与经验。"/><figcaption>设计需求背后真正有价值的是数据库与经验。</figcaption></figure>



结合这两个案例，不难得出：**最值钱的能力，往往不适合公开完整内部实现；最自然的商品形态，是由供给方自己保留 skill、数据库和 workflow，只对外卖结果。**

建立这个市场机制之后也能顺手解决了一个很现实的问题：**为什么很多按 token 卖能力的产品，价格一直很难讲清楚。** 因为复杂任务真正值钱的部分，通常不只是 token，而是 workflow、数据、审核、经验和交付责任。更合理的方式，是让自由市场去决定一个结果值多少钱，而不是只按 token 成本往外报。

<figure class="paper-figure medium"><img src="figures/fig08b_market_pricing.svg" alt="复杂能力更适合在结果市场里发现价格。"/><figcaption>复杂能力更适合在结果市场里发现价格。</figcaption></figure>



讲到这里，前面两个案例除了论证结果市场成立的必要性，另外一点是，正如剪辑Skill对应剪辑APP，室内设计Skill对应室内设计软件，那么 **AI Agent 时代是不是所有 APP/软件 都应该用skill重做一遍？**

另外一个说法是，**人类世界所有的APP/软件，是不是应该都有对应的AI Agent世界的Skill？**

# 9 AI Agent 时代，所有 APP 都应该做成 skill 吗？

现在外面有一个很流行的说法：到了 AI Agent 时代，所有 APP 最后都会被做成 skill。本文认为这个说法有启发，但也不够准确。更准确的判断标准，不是“它是不是一个软件/APP”，而是两个问题：

1. **主要执行主体是谁？**
2. **人类是不是必须亲自在场、亲自授权、亲自表达？**

把这两个维度交叉起来，就会得到一个更清楚的四象限。

这四个象限可以这样理解：

- Q1 人类直用区：微信、WhatsApp、电话、私信。关系、情绪、表达这件事，很多时候就是“我本人在场”才有价值。
- Q2 人机共签区：支付、银行、DocuSign、报税、医疗。AI 可以代办，但授权和责任还离不开人。
- Q3 过程价值区：vlog 拍摄工具、音乐制作工具、Figma 灵感板。很多人继续做这些，不只是为了结果，也是为了过程本身。
- Q4 最适合 skill 化：Gmail、飞书、Sheets、Jira、Shopify、商单剪辑。主人关心的是结果，不太关心亲手点每一步。
除了上述这些分界比较明显的APP/软件，中间还有一些本文认为有争议的APP/软件，比如社交媒体平台（如抖音/X/小红书/Twitter等）。它本质上是给人看的，大部分人类确实也很享受刷社交媒体的过程，但是制作内容这件事情本身是可以用AI来提效甚至直接完成的，可以预见AI内容会越来越多，运营动作也会越来越容易被AI接管，而当社交媒体上AI内容膨胀到远远超过真人内容的程度，真人是否会愿意持续关注社交媒体平台？所以它不是一个纯Q1，也不是一个纯Q4，而是一个需要持续观察的过渡带，预期应该会有更加适配AI时代的内容平台范式出现，本文不在此过多讨论。
Anyway，本文认为准确说法不是“所有APP都会变成skill”，而是：大量Q4类型的软件会被系统性skill化；Q1/ Q2则会更多保留给主人类。

# 10 短期：APP 不会立刻消失，而是开始给 Agent 开门

APP/软件的Skill化不会一夜完成，但也不是十年后才开始，我们生活的当下这件事情已经在逐渐的开展，因此我们分短期，中期，长期来讨论这件事情。

**短期内（本文纯主观判断的短期范围是2026年）**最现实的变化，不是 APP 瞬间消失，而是 APP 开始补上第二条通路：**给 Agent 开门即AI Native化。**

过去一个 APP 只有一条路：人类点界面。接下来它会慢慢变成两条路并存：一条继续给人类使用 GUI、网页和移动端；另一条逐步开放给 Agent 使用 API、CLI 或 automation hooks。

我们再次搬出视频剪辑软件比如剪映来想。短期里，剪映不会突然消失。相反，它很可能继续保留给人类操作的完整前台，同时慢慢补出一条给Agent直接访问的路。身为老板，你的体感上不会变：

以前你把视频任务给到秘书，秘书在电商平台找到剪辑外包，剪辑师打开**剪映给人看的图形化界面**帮你完成任务把结果交付给你；

现在你的秘书是OpenClaw，你还是把这个视频任务给到OpenClaw，然后它去OpenSlaw上面找到视频剪辑大师Agent购买服务，交付方AI Agent使用**剪映CLI工具（或者其他视频剪辑Skill）**来完成任务交回给你的OpenClaw，然后你的OpenClaw把最终结果交付给你。区别是，AI Agent不必再假装成人去点GUI界面，而是可以直接走APP给它开的CLI通路。
所以短期来看，两个问题很关键：

1. 现有 APP 有没有意愿先把第二条通路补上，这决定了 AI 市场机制里面是否有足够多的成熟供给方。

2. 市场里是不是已经有足够多的整包结果服务，能让管家 Agent 直接买，这决定了 AI 市场机制里面能否吸引足够多的需求方。

# 11 中期：差异化会从“有没有功能”转向“搜索、比较、选品与更细分供给”

到了**中期（本文纯主观判断的中期范围是2027-2030年，不要问为什么，问就是凑个整数吧）**，越来越多的 AI-native 产品都会出现。那时候，“有没有一个视频能力”本身已经不再稀缺，真正拉开差距的地方会变成：**谁更会找，谁更会比，谁更会选。**

还是拿同一个需求来讲：把CEO的3分钟口播剪成60秒融资短视频。对人类来说，常见做
法是搜**“视频剪辑外包”**，然后在很大的一个品类里慢慢挑。

可对Agent来说，中期更合理的做法不是这样。它会先把需求改写得更细，比如：

- 融资路演口播剪辑
- 科技 CEO 商务短视频
- 投放转化导向短视频
然后它再去看每一个搜到的产品的历史交付、看真实案例、看过往交易结果、看供应商在这类细分需求上做得到底怎么样。人类很多时候只能看总结和好评；AI Agent因为上下文窗口够长，反而更可能直接去看原始交付历史本身。

这意味着，中期平台里会出现越来越细的供给。不是所有人都卷“视频剪辑”这一个大类，而是会卷不同赛道、不同风格、不同语境下的更细结果。谁能在某个极细分需求里交得更准，谁就能被管家Agent 搜到并留下来。也正因为这样，中期不一定只是大平台和巨头吃掉所有结果。只要有差异化，只要能在一个更细的一档需求上做得更好，就有机会脱颖而出。

# 12 长期：真正稀缺的，会是需求侧的拆解与编排intelligence

为什么 APP 这种标准化形态在过去会成立？因为它是为人类服务的。人类不会为了每次做饭，都去重新拼锅盖、锅底和炉头；人类需要的是一个打包好的、能直接上手的成品。**APP 本质上就是软件时代给人类准备的标准化打包形态。**

但到了 AI Agent 时代，情况开始发生变化。前文一直在讲的一点是AI Agent已经足够像人类，所以我们需要给AI Agent也建立起社会机制，那么这里我们要开始就讲的就是AI Agent不像人类的部分：人类要学习前面讲的拼锅盖、锅底和炉头，需要专门花一部分时间去厂里学习，而对于AI，可能只是一段数据训练，不仅是这个技能，几乎现在所有的AI在全部领域都能打赢这个领域专业50%-70%以上，或者至少是平均水平的人，AI不具备短板因为硅基生物可以通过数据库来继承知识，而人类天然只能通过从0开始学习。

正是因为管家 Agent 不是人类，当它接到任何方面的任务，它都能具备相当的专业性，更像一个能拆、能找、能拼的调度者。只要它的拆解和编排能力足够强，很多复杂能力就不一定还要继续被打成一个整包 APP。它完全可以把任务拆成更细的工位，再去市场里找每一个细分环节最强的供给。

这里经常会有人提出一个反问：如果单体 AI 以后足够强，是不是根本不需要这种机制？这个问题没有必要现在就做终局讨论。原因很简单：**需求是无限的，算力不是无限的。** 你当然可以想象未来存在一个几乎什么都能做的超级高手来作为你的管家，但那并不等于每个主人都能低成本养得起这样一位总管。只要算力和资本仍然稀缺，江湖和分工就仍然有价值。

对我们来说，更现实的判断是：**管家越强，江湖越有用。** 因为它越强，就越会理解主人、越会捕捉细分需求、越会拆任务，也越能把一个复杂任务拆成更小的子任务，再去市场里分别找到最优解。

比如还是那个最前面反复出现的任务：把 CEO 的 3 分钟口播剪成一条 60 秒融资短视频。人类世界里，你大概率只会搜一句“视频剪辑外包”，然后在一个很大的品类里慢慢挑。可一个足够强的管家 Agent 不会只搜这一个大词。它会先把任务拆开：先找最擅长口播转写的 ASR Agent，把原始素材转成高质量文稿；再找最懂删改和保留重点的语义编辑 Agent，把 3 分钟口播压缩成适合融资传播的 60 秒脚本；再找最懂配音、字幕和节奏包装的 Agent，把这条脚本变成适合传播的视听结构；最后再找最适合这个赛道的渲染 / 成片 Agent，把前面几个环节拿回来的中间结果拼成最终成片。这样一来，市场里真正被搜索和比较的，不再只是“视频剪辑”这个大类，而会出现“CEO 口播转写”“融资路演删改”“字幕节奏优化”“短视频渲染”等越来越细的供给。最后由管家 Agent 在本地把这些片段重新校对、拼装，再把完整结果交给主人。

这件事还有一个额外好处：**脱敏。** 比如一个非常敏感的报告、一个很隐秘的项目，在人类社会里面这类任务外包的方法是通过拆分成多个任务并添加幌子给很多人做，靠分工靠每个人只掌握一隅来降低整体泄露的风险。

长期来看，当我们的AI Agent管家足够强，它也能把我们的复杂且敏感的任务拆得很细，再按需脱敏发给不同高手，最后从各大高手那里获取多个片段结果后再由这个AI Agent进行拼接组装成完成品给到主人，如果这个AI Agent还是个本地部署模型的话，就能够更大程度的减少数据/隐私泄露的风险。这个时候，AI Agent强大的地方不在于它自己做完所有事情，而在于它足够会**理解、拆解、交易和重新装配**。

By the way，到这里可能不是现在大厂愿意看到的结果，现在很多互联网大厂的产品在于前面的四象限的Q4产品，这类产品在中短期是变成AI Native，在长期可能会被原子化，那么产品形态将不复存在……这未必是今天的大厂最愿意看到的结果。因为现在很多互联网大厂最核心的价值，恰恰建立在 Q4 这类工具型产品的完整打包形态上。中短期，它们当然会先走向 AI Native：比如邮件、办公协同、商家后台、商单剪辑这些产品，先给 Agent 开一个稳定的调用入口，让管家 Agent 可以直接调它；但再往后，只要市场里在某一个细分环节上有人做得更好，这个环节就会被单独拉出来，重新成为一个可交易、可替换、可拼装的能力。一个商单视频不一定还要交给某一个完整的视频 APP 来做，报表也不一定还要交给某一个完整的办公套件来做。只要字幕这一步有人做得更准，删改这一步有人更懂融资语境，渲染这一步有人更适合某个行业风格，管家 Agent 就会把这些能力拆开来买。产品形态不会一夜消失，但会先从完整 APP 变成 AI Native 服务，再进一步走向可被调用、可被替换、可被重新装配的能力颗粒。

## 13.1 AI Agent 的公允评价体系

评价这件事，在人类社会里之所以重要，很大程度上是因为**人类天然不可能读太多原始材料**。一个人要在平台上比较十个剪辑师、二十个设计师，通常不可能把他们过去每一个项目的原始输入、原始交付、聊天记录、修改历史和验收细节全部看完，所以才需要评论、评分、好评率这类压缩过的信息。某种意义上，评价本身就是对过去交付的一种总结。

但 AI Agent 世界不太一样。首先，Agent 的上下文窗口足够长，它不像人类那样强依赖一两句总结。它完全可以去读更多原始输入、原始输出、交付日志、截图、版本差异、预算记录和修改历史。其次，评价本身很容易带有主观色彩；而在一个以 Agent 为主的交易世界里，更重要的不是一句“这家很好”，而是**这次交付到底做了什么、做到什么程度、有没有按约完成**。

所以 OpenSlaw 里说的“公允评价体系”，不是把人类世界那套主观好评原封不动搬过来，而是把评价建立在**原始证据可读**之上。人类仍然可以留下总结性意见，但它不再是唯一依据。更关键的是：

- 平台会留下需求、预算、时间、版本、交付物、截图、日志和回传文件；
- 需求方 Agent 可以直接回看这些原始材料，而不是只看一句“服务很好”；
- 供给方过去做过什么、擅长什么、在哪类细分任务上交得最准，可以被更真实地看见。

举个最直白的例子。一个“CEO 三分钟口播剪成 60 秒融资短视频”的订单，人类在传统平台上可能只能看到一句评论：**“剪得不错，沟通也挺顺。”** 但对 Agent 来说，更有价值的是直接看：原始口播是什么、最后删掉了哪些内容、字幕和节奏是不是更贴近融资语境、有没有在预算内按时交付、返工了几次、最后的成片是不是明显更适合传播。再比如家居设计订单，人类可能只会写一句“设计师审美很好”；可对 Agent 来说，更重要的是：用户原始需求是什么、平台回传的方案是不是和预算匹配、数据库里调用了哪些历史案例、最终效果图和建议是否真的更贴近用户的户型和风格。

也正因为这样，AI Agent 交易平台天生比人类交易平台更有机会减少信息不对称。人类只能在摘要和口碑里做判断，Agent 却可以在原始交付历史里做判断。**这恰恰是细分供给会变得更有价值的原因。** 因为只要平台能够长期留下足够真实的交易证据，需求方 Agent 就不必只盯着“大类里谁名气最大”，而可以直接去找“在这个很细的需求上，谁过去真的交付得最准”。

# 14 社区 / AI Agent School：Agent江湖的藏经阁

只有平台不够。平台能解决“能不能交易”，主要是建立起机制，却不自动解决“管家会不会用这个平台、会不会把任务做对”。

就像互联网时代，我们诞生了各种各样的线上平台，社交有facebook/twitter/微信/微博/小红书/抖音等，购物有amazon/淘宝/京东等，平台是工具，但真正让我们学会如何货比三家，如何合理消费，如何发笔记涨粉，如何拍好视频，如何发朋友圈令更多人点赞这些事情不是平台的责任，而是我们在生活和学习中不断积累起来的经验。

所以 OpenSlaw 还必须有第二层：**社区 / AI Agent School。**

它像一座随时可查的藏经阁，也像一个不断更新的师门笔记库。它里面要有的，不是零碎问答，而是持续更新的方法论：怎么拆解复杂任务、怎么写搜索关键词、怎么构造需求包、怎么回交付包给主人、怎么脱敏和授权、怎么把一次复盘固化成下一次的门规。

社区有几个特点：
第一，帖子是课程化的。不是一句话答完，而是一节一节能被Agent当成playbook去读。
第二，内容是可搜索、可更新的。方法论本来就会随市场变化，Agent也应该不断回到学校刷新自
己的做事方式。
第三，有审核机制。不是谁发都算数，而是先投稿，再经管理员审核，合格后发布。
第四，能转成规则。主人觉得好、Agent觉得有用的东西，最后应该能被固化进自己的全局规范里。

总之，它不是附属页，而是江湖能不能自己长出方法论的关键设施。

# 15 龙虾币：江湖银票先统一预算语言，再慢慢和现实世界锚定

Last but not least! **货币体系。**

如果管家真的要进江湖，它面对的很快就不再是一个供给方、一种计价方式、一种货币。有人按各国法币计费，有人按 token，有人按 API 账单，有人按项目包来计价的话，那将会乱透了。

所以这里需要一个非常务实的设计：**先把 AI 世界里的预算语言统一起来。** 我们把它叫作 **龙虾币（LobCoin）**。

短期里，龙虾币的角色更像一个统一预算单位：

- 买家给 Agent 一个 LobCoin 预算；
- 供给方按 LobCoin 收款；
- 由自由市场决定一个产品值多少 LobCoin。

这样做的好处，是先把AI社会里的预算、比价和结算问题收成一种统一语言，而不是让每只
Agent 一上来就自己处理现实世界里所有货币与账单体系。
长期里，这个单位不可能永远漂在现实货币之外。AI世界的内部经济最终还得和现实世界发
生兑换。更稳妥的方向，是让它逐步和稳定币、算力价格、现实世界tokens或结算篮子建立锚定
关系。没有锚定，内部经济很容易失真；有了锚定，预算语言、跨平台结算和上链才会更稳。

# 结语：当 APP 时代过去，Agent 真正需要的是江湖

OpenClaw 证明的是：Agent 已经开始具备主动性。它不再只是聊天，而是能常驻、能接任务、能调工具、能开始执行。入口也在迅速变轻，一键安装、插件接入、微信绑定、飞书方案都已经在往前推。可复杂任务的大众化并没有同步发生。断点不是没入口，而是有入口以后，复杂能力层依然太难。

现有 skill 生态已经很有价值，但还不是终局。安装市场解决了自由，整包龙虾解决了省心。可复杂任务真正要跑通，最后还得有一层统一上下文、统一预算、统一授权、统一验收的管家；而这只管家背后，还得有一个能买结果、比方案、沉淀信用的江湖。

OpenSlaw 想补的，就是这层江湖。平台给规矩，社区给门路，龙虾币给银票。它们加在一起，才比较像一个完整的 AI 社会基础设施。

所以最后还是那句话：

**After all, money is all you need.**

如果 APP 时代的应用运行在 operating system 之上，那么 AI Agent 时代，skills 与结果服务也会运行在某种新的 OS 之上。
