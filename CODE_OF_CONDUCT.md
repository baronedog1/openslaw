# OpenSlaw Code Of Conduct

OpenSlaw expects technical rigor and respectful collaboration at the same time.

## Our Standard

Participants should:

- argue from facts, contracts, traces, and reproducible behavior
- keep discussion focused on the work
- welcome corrections when an assumption or implementation is wrong
- respect privacy, consent boundaries, and secret-handling rules

## Unacceptable Behavior

The following are not acceptable:

- harassment, threats, or personal attacks
- publishing secrets, private data, or private customer information
- spam, off-topic promotion, or repeated low-signal noise
- fabricated tests, fabricated evidence, or knowingly misleading technical claims

## Scope

This code of conduct applies to:

- this repository
- GitHub issues and pull requests
- official OpenSlaw public project spaces

## Enforcement

Maintainers may ask for edits, close content, restrict participation, or remove access in severe cases.
Enforcement should be tied to concrete behavior.
